## What is v3.6?

**v3.6 = Static-First Architecture**

Instead of running a server to generate data on-demand:
- ‚úÖ Pre-compute everything offline (build step)
- ‚úÖ Generate static JSON files
- ‚úÖ Serve instantly (no database, no API)

**Result:** Zero infrastructure, instant performance, $0/month cost.

## Understanding Verification

### What does `verified: true` mean?

A product is "verified" when it's matched with the Halilit catalog:
- Name similarity: ‚â•85%
- Confidence score: 0.0 to 1.0
- Match quality: "excellent" (‚â•95%) or "good" (‚â•85%)

### Example: Nord Drum 3P

```json
{
  "name": "Nord Drum 3P",
  "verified": true,                    // ‚úÖ Found in Halilit
  "verification_confidence": 0.9714,   // 97.14% match
  "match_quality": "excellent",        // Top tier match
  "halilit_sku": null                  // Not in catalog yet
}
```

### Why is verification rate low (2.5%)?

Many brands don't have Halilit catalog data yet:
- **Adam Audio**: 100% verified ‚úÖ

- **M-Audio**: 8.3% verified ‚ö†Ô∏è

## Troubleshooting

### Build fails with "module not found"

```bash
# Make sure you're in the right directory
cd /workspaces/hsc-jit-v3/backend
python build.py --brand=nord
```

### Output directory not found

```bash
# Create frontend public/data directory
mkdir -p /workspaces/hsc-jit-v3/frontend/public/data
```

### Verification rate is 0%

Check if brand has Halilit catalog:
```bash
# Look for brand-specific Halilit data
ls backend/data/catalogs_halilit/ | grep nord
# Should show: nord_halilit.json

# Check if it has products
cat backend/data/catalogs_halilit/nord_halilit.json | jq '.products | length'
```

### Script permission denied

```bash
# Make scripts executable
chmod +x v3.6_commands.sh
chmod +x backend/build.py
```
